# Roadmap

## Phase 1 - Foundations

- Install Ubuntu Server on PC1 and PC2.
- Configure stable Host-Only networking.
- Install Pi-hole on PC1.
- Install Docker and Portainer on PC2.
- Deploy n8n and Uptime Kuma.

## Phase 2 - Secure Access

- Install WireGuard.
- Configure server and client profiles with placeholders.
- Validate VPN handshake.
- Confirm DNS resolution through Pi-hole over VPN.

## Phase 3 - Automation

- Fail2Ban alert workflow.
- Uptime Kuma health workflow.
- Backup workflow with encryption.
- Network scan workflow.
- DNS statistics workflow.
- Daily executive report workflow.

## Phase 4 - Visualisation

- Build a dashboard with service status.
- Add DNS and uptime indicators.
- Add security event summaries.
- Prepare screenshots with sensitive data hidden.

## Phase 5 - Testing and Demo

- Controlled SSH brute-force simulation.
- Controlled service outage test.
- DNS filtering test.
- Backup and restore verification.
- Final demo recording.
